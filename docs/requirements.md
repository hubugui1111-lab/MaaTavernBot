# 项目需求文档

## 项目名称

炉石传说酒馆战旗自动化脚本（Hearthstone Battlegrounds Bot）

## 项目目标

基于 MaaFramework 框架，开发一款运行在 Windows 电脑上、通过 ADB 连接 MuMu 模拟器的炉石传说酒馆战旗自动化脚本。

## 目标用户

- 不懂代码的普通玩家
- 需要简单易用的图形界面
- 一键启动/停止

## 功能需求

### FR-1: 设备连接
- 自动检测并连接 MuMu 模拟器（ADB 端口 7555）
- 连接状态实时显示

### FR-2: 游戏启动与导航
- 自动启动炉石传说
- 自动导航至酒馆战旗模式
- 自动寻找对局

### FR-3: 英雄选择
- 固定选择最左边的英雄
- 确认选择并进入对局

### FR-4: 招募阶段操作
- FR-4.1: **升级酒馆** — 有钱就优先升级（策略：优先升级 > 购买随从）
- FR-4.2: **购买随从** — 遍历酒馆随从位，尝试购买（不刷新酒馆）
- FR-4.3: **打出手牌** — 遍历手牌位，尝试全部打出（不识别卡牌内容）
- FR-4.4: **出售随从** — 手牌打不出时，出售场上最右边的随从
- FR-4.5: **英雄技能** — 金币花完后使用一次
- FR-4.6: **结束回合** — 完成操作后点击结束回合

### FR-5: 目标选择策略
- 战吼/法术需要选择目标时 → 固定选择场上中间位置的随从
- 只需尝试点击目标位置即可

### FR-6: 战斗阶段
- 等待战斗自动进行，不做任何操作
- 检测战斗结束后进入下一招募阶段

### FR-7: 对局生命周期
- 检测对局结束（排名结算画面）
- 自动退出结算
- 自动重新寻找对局（无限循环）
- 手动点击停止按钮可终止

### FR-8: UI 界面
- 主色调：褐色/棕色系
- 简洁布局
- 开始/停止按钮
- 实时状态显示（回合数、金币、当前阶段）
- 详细操作日志

## 非功能需求

### NFR-1: 易用性
- 无需编程知识即可运行
- 清晰的错误提示（如设备未连接）

### NFR-2: 稳定性
- 单步操作失败不影响整体流程
- 异常状态自动恢复（如意外弹窗）
- 操作间隔合理（避免过快点击）

### NFR-3: 可维护性
- 模板图片独立存放，方便更新
- 坐标参数集中管理
- 清晰的项目结构

## 用户确认的策略选项

| 问题 | 选择 |
|------|------|
| 英雄选择 | 固定最左边 |
| 升级时机 | 有钱就升 |
| 出售时机 | 打不出牌时卖最右边随从 |
| 酒馆刷新 | 不刷新 |
| 英雄技能 | 金币花完后用一次 |
| 冻结酒馆 | 不冻结 |
| UI 内容 | 开始/停止 + 状态 + 日志 |
| 自动重开 | 无限循环 |
